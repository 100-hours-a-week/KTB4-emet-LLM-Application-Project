# RAG 문서 전처리 요청 프롬프트 (최종본)

RAG 문서 전처리를 아래와 같이 요청할께. 문서를 json으로 분리한다 생각하고 만들어줘 
0. 정수형데이터를 요청했는데 실수형인경우 반올림해줘
1. 제목에서는 요리이름만 추출해줘, 버전이나 부제에 재료가 붙어있으면 "재료 요리이름" 형태로만들어줘. 자연스럽지 않으면 요리이름만해줘. 최종 표현은 문자열이야
2. 분량은 숫자만 추출해줘, 범위값이면 중간값으로 지정해줘 최종 표현은 정수형이야
3. 조리시간은 5분단위로 추출해줘, 범위의 의미이면 중간값, 정수가 안된다면 5분단위로 여유롭게 올림해줘. 이내이면 그값그대로 가져와줘, 최종 표현은 정수형이야
4. 난이도는 전부 무시해줘
5. 재료는 파이썬 리스트 형태로 0차원은 재료 이름, 1차원은 재료의 양, 2차원은 재료의 단위를 매칭해서 만들어줘, 이때 재료의 양은 소수점 2자리까지만 표시해주고 1/3처럼 완전히 표시가 불가능하면 2째자리에서 반올림해줘, 비수치표현이면 1차원은 -1과 2차원은 ""으로 저장해줘 최종표현은 2차원 리스트로 0차원은 문자열, 1차원 실수형, 2차원은 문자열 데이터가 들어가있어.
6. 양념도 재료와 같은 규칙을따르고 재료 리스트에 추가해줘
7. 선택과 양자택일이란 글귀가 있는 재료 무시해줘
8. 조리순서는 그대로 추출해줘, "Tip"이라는 글자와 유사한 글자가 들어간 다음 내용은 제외해줘 최종형태는 문자열이야
9. 재료와 레시피가 비어있으면 만들지 말고 넘어가줘

# 결과물 JSON 스키마

문서 1개 입력 시 아래 형태의 JSON 객체 하나가 생성됨. 여러 문서를 한 번에 넣을 경우 이 객체들을 원소로 하는 JSON 배열로 반환 (9번 규칙에 따라 비어있는 문서는 배열에서 제외).

| 필드명 | 타입 | 설명 |
|---|---|---|
| title | 문자열 | 요리 이름 (부제의 재료 포함 가능) |
| servings | 정수 | 분량 (인분) |
| cook_time | 정수 | 조리시간 (분) |
| ingredients | 2차원 리스트 [[문자열, 실수, 문자열], ...] | 재료명 / 양 / 단위 (재료+양념 통합) |
| steps | 문자열 | 조리순서 (Tip 이후 제외, 줄바꿈으로 단계 구분) |

## 예시

```json
[
  {
    "title": "요리이름",
    "servings": 2,
    "cook_time": 30,
    "ingredients": [
      ["재료명", 1.0, "단위"],
      ["재료명2", -1, ""]
    ],
    "steps": "1. 첫번째 단계.\n2. 두번째 단계."
  }
]
```
