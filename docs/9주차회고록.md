## 차주 목표
### 요리레시피(문서) 정형화
  - pdf파일 그 자체를 쓰기에는 레시피 작성자가 전부 달라서 통일성 문제가 발생한다. 그래서 pdf레시피를 읽고 정형화하는 프롬프트를 만들어서 LLM 에게 처리시켜 통일성있는 json형태로 저장할려고한다.
  - 예상 결과물: 정형화 AI 에이전트(레시피(pdf) -> 데이터 전처리 프롬프트 부착 -> LLM 정형화 요청 -> 정형화 레시피(json)) (요리이름, 재료분할, 조리시간, 난이도, 조리법)
    
### 질의에서 요리재료만 추출
  - 주요관점은 해당 어플리케이션의 주요 정보는 현재 요청한 "요리재료"이기에 LLM을 이용해서 질의에서 "요리재료"를 추출해서 어플리케이션에 내에서 여러 형태로 사용할 수 있을 것이다.
  - 예상 결과물: 재료 추출 노드 





## 회고
개인 사유 때문에 남들보다 시간이 적은 저에게 alex의 RAG 프로젝트는 저에게 도움이 많이 됬습니다. 이론적인 측면은 다행이 항상 당일에 복습을 하기때문에 문제가 없었는데 직접개발하는 측면에서 문제가 있엇습니다. 하지만 코드를 읽으면서 이해하고, 중간 중간 저의 형태로 변경해가면서 제코드 형식으로 녹였습니다. 그리고 교재 및 정리한 내용을 다시 보면서 개인 프로젝트에 적합하도록 일부 수정도 진행했습니다. 다음 주차 목표는 alex 프로젝트 안보고 개인 능력으로 직접 랭그래프로 마이그레이션, 실행 마다 매번 불러오는 다큐먼트 및 임베딩 모델 횟수또는 리소스 줄이기 입니다.
