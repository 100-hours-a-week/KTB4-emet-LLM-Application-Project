## 차주 목표
### 요리레시피(문서) 정형화
  - pdf파일 그 자체를 쓰기에는 레시피 작성자가 전부 달라서 통일성 문제가 발생한다. 그래서 pdf레시피를 읽고 정형화하는 프롬프트를 만들어서 LLM 에게 처리시켜 통일성있는 json형태로 저장할려고한다.
  - 예상 결과물: 정형화 AI 에이전트(레시피(pdf) -> 데이터 전처리 프롬프트 부착 -> LLM 정형화 요청 -> 정형화 레시피(json)) (요리이름, 재료분할, 조리시간, 난이도, 조리법)
    
### 질의에서 요리재료만 추출
  - 주요관점은 해당 어플리케이션의 주요 정보는 현재 요청한 "요리재료"이기에 LLM을 이용해서 질의에서 "요리재료"를 추출해서 어플리케이션에 내에서 여러 형태로 사용할 수 있을 것이다.
  - 예상 결과물: 재료 추출 노드 

## 회고
금요일 휴가때 지도교수님 생신차 연구실 인원들을 만나면서 현재 내 개인프로젝트와 연관되어 현업하고있는 자들에게 지식을 공유받았다. LLM이 가장 잘하는것은 결국 텍스트를 처리하는 것이다. 그러니 질의 그자체를 사용하기보다는 질의를 분석하고 해체하는 방식으로 사용하는 것도 일종의 사용법이라는 조언을 받았다. 해당 조언을 통해 아이더의 구상 폭이 넓어지면서 플로우차트를 작성하고, Claude와 대화를 하면서 내생각을 선명하게 만들어 갔다. 그 결과 1차 초안이 만들어졌으며, 이를 토대로 기존 코드를 대대적으로 수정을 진행중이다.
