### 요리레시피(문서) 정형화
  - pdf파일 그 자체를 쓰기에는 레시피 작성자가 전부 달라서 통일성 문제가 발생한다. 그래서 pdf레시피를 읽고 정형화하는 프롬프트를 만들어서 LLM 에게 처리시켜 통일성있는 json형태로 저장할려고한다.
  - 예상 결과물: 정형화 AI 에이전트(레시피(pdf) -> 데이터 전처리 프롬프트 부착 -> LLM 정형화 요청 -> 정형화 레시피(json)) (요리이름, 재료분할, 조리시간, 난이도, 조리법)
### 질의에서 요리재료만 추출
  - 주요관점은 해당 어플리케이션의 주요 정보는 현재 요청한 "요리재료"이기에 LLM을 이용해서 질의에서 "요리재료"를 추출해서 어플리케이션에 내에서 여러 형태로 사용할 수 있을 것이다.
  - 예상 결과물: 재료 추출 노드

## 차주 목표
### 레시피 생성
  - LLM이 이전 노드에서 정형화한 재료 정보를 토대로 레시피를 생성하는 prompt 작성 및 노드 개발
### 레시피 검토
  - LLM이 레시피 생성 후 식재료,조리법의 이상, 요리자체의 이상 등을 판별하는 prompt 작성 및 노드 개발
### 레시피 리스트 통합 및 응답
  - RAG를 통해 찾은 레시피와 LLM이 생성한 레시피를 하나의 리스트롤 통합해서 응답하는 노드 개발

## 회고
지난주 네트워킹데이와 주말간 몸상태 악화로 차질이 벌어져서 이번주 목표와 해당 회고록 작성에 지연이생겼습니다. 일단 이부분에 대해서는 좀더 시간을 타이트하게 가져서 출퇴근시간에 프롬프트 엔지니어링을 공부해서 개발 시간을 줄이는것을 목표로 할려고 합니다. 이번주와 차주의 내용이 대부분 prompt작성이 메인인 개발이다보니 프롬프트 엔지니어링의 중요성에대해 알 수 있게되었습니다.
